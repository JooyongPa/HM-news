## 3. 매일 하는 일 — 발행 절차

주용파님이 "픽5 발행해줘" / "대한뉴스 발행해줘" 라고 하면 아래를 수행한다.

### 3-1. 재료 찾기
오늘 날짜의 폴더를 `동영상 뉴스` 안에서 찾는다. 없으면 만들지 말고 물어볼 것.
(실제로 날짜를 착각하신 적이 있다. 7/15 것을 7/16에 말씀하셔서 하루 밀린 채로
반영된 적이 있으니, 표지 이미지의 날짜와 오늘 날짜가 맞는지 꼭 대조할 것.)

### 3-2. 제목 뽑기
표지 이미지(`pick5_01.png` 또는 영상 코너의 `0-1.png` 등)를 **직접 읽는다.**
이미지 안에 이런 식으로 적혀 있다.

```
2026.07.21 · 주용파 PICK 5
오늘의 핵심 뉴스 5선
WAICO 창설 · 원화 국제화 · 이란 봉쇄 유가 급등     ← 이 줄이 제목
```

부제 줄(가운뎃점으로 나열된 줄)을 그대로 제목으로 쓴다. 임의로 요약하거나
바꾸지 말 것. 영상 코너(대한뉴스 등)는 표지에 부제가 없을 수 있는데,
그럴 땐 02~04번 이미지의 헤드라인들을 가운뎃점으로 이어서 만든다.

예: `한은 기준금리 3년반 만에 인상 · 코스피 7천선 재탈환 · SK하이닉스 나스닥 상장`

### 3-3. 표지 이미지 넣기
```
thumbs/YYYY-MM-DD_<코너>.png
```
코너 이름은 `pick5` / `digital` / `daehan` / `gompapa` 중 하나.

**압축할 것.** 아래 4-3 참고. 지금 저장소 용량의 88%가 압축 안 한 표지들이다.

### 3-4. index.html 고치기
`POSTS` 배열 **맨 위**에 새 항목을 끼워 넣는다. 반드시 맨 위여야 한다(아래 4-1 참고).

```js
const POSTS = [
  { date: '2026-07-21', cat: 'pick5', title: 'WAICO 창설 · 원화 국제화 · 이란 봉쇄 유가 급등',
    insta: 'https://www.instagram.com/reel/DbCEZcxN0B4/',
    threads: 'https://www.threads.com/@jooyongpa/post/DbCEb4Hk7mj',
    blog: 'https://blog.naver.com/act2lab/224352816791',
    thumb: 'thumbs/2026-07-21_pick5.png' },
  ...
```

**URL은 물음표 뒤를 잘라서 넣는다.** 주소.txt에 있는
`?utm_source=ig_web_copy_link&igsh=...` 같은 추적 파라미터는 제거.

**제목에 작은따옴표가 들어가면 반드시 이스케이프.** 7/18 곰파파 때 `'모두의 AI'`
때문에 JS 문법이 깨져 사이트 전체가 백지가 될 뻔했다. 넣은 뒤 문법 검사할 것.

### 3-5. OG 태그 고치기 (index.html `<head>`)
```html
<meta property="og:title"  content="주용파의 오늘도 수고했어">        ← 영구 고정
<meta property="og:url"    content="https://hm-news.vercel.app">      ← 영구 고정
<meta property="og:description" content="PICK 5 — (오늘 제목)">        ← 매일 교체
<meta property="og:image"  content="https://hm-news.vercel.app/thumbs/YYYY-MM-DD_코너.png">  ← 매일 교체
```

description 앞머리는 코너 이름 + ` — `.
`PICK 5 — ` / `디지털 브리핑 — ` / `대한뉴스 — ` / `이번 주도 수고했어 — `

제목과 주소를 고정하는 건 주용파님이 직접 정하신 방침이다. 매일 전부 바꾸는 건
품이 너무 든다고 하셔서, 아랫줄 두 개만 바꾸는 쪽으로 합의했다.

### 3-6. 올리기
```bash
git add -A
git commit -m "2026-07-21 PICK5"
git push
```
Vercel이 자동 배포한다.

### 3-7. 확인하고 링크 드리기
배포되면 1면에 오늘 것이 리드로 떴는지 확인한다.
그리고 **캐시 우회 링크**를 만들어 드린다.

```
https://hm-news.vercel.app/?v=20260721
```

카톡·스레드는 URL마다 미리보기를 캐시해두기 때문에, 같은 주소로 다시 공유하면
어제 카드가 뜬다. 뒤에 `?v=날짜`를 붙이면 새 주소로 인식해서 오늘 카드가 뜬다.
사이트는 이 값을 무시하므로 아무 문제 없다. 주용파님이 이 링크를 각 채널에
다시 뿌리시므로 **발행할 때마다 챙겨서 드릴 것.**

하루에 두 번 발행하는 날(픽5 + 대한뉴스)은 `?v=20260716b` 처럼 뒤에 글자를 붙인다.

## 4. index.html 구조

### 4-1. 정렬과 1면 배치 — 중요
```js
const sorted = [...POSTS].sort((a, b) => b.date.localeCompare(a.date));
```
날짜 내림차순. **JS sort는 안정 정렬이라 같은 날짜면 배열에 먼저 있는 게 앞선다.**
그래서 같은 날 두 건을 발행하면 나중에 넣은 게 리드가 되려면 배열 맨 위여야 한다.
새 항목을 항상 맨 위에 끼워 넣는 이유가 이것이다.

1면 배치 로직:
```js
const lead = sorted[0];              // 가장 최신 = 리드
const seen = new Set([lead.cat]);
const side = [];
for (const p of sorted.slice(1)) {
  if (!seen.has(p.cat)) { side.push(p); seen.add(p.cat); }  // 코너 중복 제외
  if (side.length >= 3) break;
}
```
사이드에는 리드와 **다른 코너**의 최신 글이 3개 뜬다. 그래서 1면에는 항상
네 코너가 하나씩 보인다. 요일과 무관하게 자동으로 정해지니 손댈 필요 없다.

### 4-2. 코너 정의
```js
const CATS = {
  pick5:   { name: 'PICK 5' },
  digital: { name: '디지털 브리핑' },
  daehan:  { name: '대한 뉴스' },
  gompapa: { name: '이번 주도 수고했어' },
};
```
색상은 CSS 변수로 코너마다 지정돼 있다(`--pick5-bg` 등). 새 코너를 만들 게 아니면
건드릴 일 없다.

썸네일 클릭 시 열리는 링크는 `insta → threads → blog` 순으로 있는 것을 쓴다.
```js
function mainLink(p) { return p.insta || p.threads || p.blog || ''; }
```
채널 버튼은 값이 있는 것만 렌더링되므로, 링크가 없으면 그 항목은 비워둬도 된다.

### 4-3. 이미지·영상 처리

**표지 이미지 압축 — 꼭 할 것.**
현재 저장소 12.4MB 중 11MB가 압축 안 한 표지 5개다.
```
thumbs/2026-07-11_gompapa.png   2,509,003
thumbs/2026-07-18_gompapa.png   2,485,642
thumbs/2026-07-14_digital.png   2,098,894
thumbs/2026-07-09_daehan.png    2,034,191
thumbs/2026-07-16_daehan.png    1,900,472
```
영상 코너 표지를 원본 그대로 넣어서 그렇다. 화면에선 작게 보이는 썸네일인데
2MB짜리를 받아오고 있다. 폭 1080px, 품질 80 정도의 JPG로 줄이면 100~200KB면 된다.
눈으로는 차이가 안 난다.
```bash
ffmpeg -i 원본.png -vf scale=1080:-1 -q:v 4 thumbs/2026-07-23_daehan.jpg
```
픽5 표지는 원래 45KB 안팎이라 그대로 둬도 된다.
기존 5개도 여유 있을 때 교체하면 좋다(다만 git 기록에는 남아 용량이 줄지는 않는다).

**영상은 색인을 앞으로 옮길 것.**
픽5 영상은 `moov` 원자가 파일 끝에 있어서, 브라우저가 통째로 받을 때까지
재생을 못 하고 멈춘다. 실제로 이것 때문에 하루 종일 헤맸다.
```bash
ffmpeg -i 원본.mp4 -c copy -movflags +faststart 결과.mp4
```
이러면 색인이 파일 앞(36바이트 위치)으로 온다. 재인코딩이 아니라 재포장이라
화질 손실도 없고 1초면 끝난다.

### 5-3. 저장소 용량 정책
- 픽5 영상(약 1.9MB/편)은 저장소에 넣어도 된다. 연간 약 500MB.
- **대한뉴스·곰파파 영상은 넣지 말 것.** 7/16 대한뉴스가 117MB였다. 픽5 60일치다.
  이 둘은 계속 인스타를 저장소로 쓴다.
- 현재 저장소 12.4MB. GitHub이 편한 선이 1GB 정도이므로 2년쯤 여유.
- git은 지운 파일도 기록에 남아 용량이 줄지 않는다. 1년 뒤 정리 시점에 알려드릴 것.

## 6. 아직 안 끝난 것 — 픽5 영상 자체 재생

### 배경
지금은 1면 썸네일을 누르면 인스타로 나간다. 이걸 신문 안에서 바로 재생되게
바꾸는 걸 테스트했고, **성공했다.** 맥과 휴대폰 양쪽에서 확인 완료.

시행착오 기록:
1. 구글 드라이브 iframe → 재생은 되나 모바일에서 구글 UI가 화면을 덮음. 탈락.
2. 드라이브 파일 직접 재생 → 구글이 막아둠. 탈락.
3. **저장소에 영상을 직접 넣고 `<video>` 태그** → 성공. 이 방식으로 간다.

테스트 페이지가 아직 저장소에 남아 있다: `test-drive.html`, `videos/pick5_2026-07-20.mp4`
정리하거나 본 적용에 활용할 것.

### 이렇게 되면 좋은 점
발행이 인스타 업로드를 기다릴 필요가 없어진다. 06시에 픽5가 생성되면
바로 영상까지 포함해 발행할 수 있고, 인스타·스레드·블로그 링크는
나중에 주용파님이 올리신 뒤 붙이는 "덤"이 된다. 안 붙여도 신문은 멀쩡하다.

### 목표 흐름
```
06:00  코드가 픽5 생성
06:00  코드가 영상 faststart 처리 + 표지 압축 + 저장소 push → 신문 자동 발행
06:00  발행된 신문 주소를 주용파님께 메일 발송
```

## 7. 대한뉴스·곰파파는 어떻게 하나

주용파님이 웹에서 제작해 완성본을 넘기신다. **방식은 지금과 동일하게 유지**하기로
결정했다(주용파님 승인).

1. 주용파님이 `동영상 뉴스` 안에 날짜 폴더를 만들고 표지 이미지와 `주소.txt`를 넣는다
2. 인스타·스레드·블로그에 직접 업로드하고 주소를 `주소.txt`에 적는다
3. "대한뉴스 발행해줘" 라고 말씀하신다
4. 코드가 3장 절차대로 발행한다

바뀌는 건 말할 상대뿐이다. 영상은 저장소에 넣지 않고 인스타 링크를 쓴다.
